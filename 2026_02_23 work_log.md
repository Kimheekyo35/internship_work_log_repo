## linuxuser가 root로 바뀌는 오류
💢가장 큰 이유는 `sudo chown -R 50000:0 /home/linuxuser/wemarketing/메타광고현황수집` 때문에 airflow와 root만 작성할 수 있는 형태로 변환하기 때문임

그럼 airflow와 root, linuxuser의 id는 어떻게 아는 것❔
`docker exec -it airflow id` 를 하면 airflow가 50000인 것을 알 수 있음
`id linuxuser` 를 하면 1001 인 것을 알 수 있음

### chown -R 50000:0 의미
`chown -R 50000:0`
50000 → 소유자 UID
0 → 그룹 GID (0은 root 그룹)
-R → 하위 폴더 전체 적용

즉, 해당 폴더의 주인(owner)을 UID 50000으로 변경하고 그룹을 root 그룹으로 설정하여 “주인”은 UID 50000

### chmod -R 775 의미
▶️ 권한 설정
775는 아래와 같이 읽음
```
7   7   5
|   |   |
u   g   o

u = user(소유자)
g = group(그룹)
o = other(그 외 모든 사람)
```

숫자의 의미
숫자의미 
- 4 - read (r) 
- 2 - write (w)
- 1 - execute (x)
더해서 표현함.
7 = 4 + 2 + 1 = rwx
5 = 4 + 1 = r-x

그래서 775는?
- 대상권한소유자(50000)rwx (읽기/쓰기/실행)
- 그룹(root)rwx
- 기타 사용자r-x (읽기/실행만, 쓰기 불가)

따라서 linuxuser가 소유, 그룹을 root로 하고 싶으면
`sudo chown -R linuxuser:root /home/linuxuser/wemarketing/어쩌고` 한 후, `sudo chmod -R 775` 주면 됨.
