# 2026_03_26 WORK_LOG

## Docker 사용

현재 Airflow를 컨테이너화하여 배포하고 있는데 `docker-compose.yml` 이나 `docker-compose.volumes.yml` 를 새롭게 수정하여 up -d 할 떄마다 airflow의 dag 설정이 번거로웠음.

**Docker 관련하여 수정한 것**: 
- `docker compose exec airflow airflow config get-value core dags_are_parsed_at_creation` 에서 `dags_are_parsed_at_creation`이 True 일 경우, 새로 build 하고 나서 dag들이 중지됨
- `docker-compose.yml` 에서 environment 부분에 `AIRFLOW__CORE__DAGS_ARE_PAUSED_AT_CREATION: "false"` 해당 부분 추가하면 다시 컨테이너를 동작시켰을 때 원래대로 실행됨.

수정한 후, `docke-compose.yml` 만 수정하여 `docker compose -f docker-compose.yml up -d` 만 진행하였더니 volumes 설정이 제거되어 dags들이 airflow에서 제거되는 문제점 발생함.
- 이럴 경우, `docker compose -f docker-compose.yml -f docker-compose.volumes.yml up -d` 진행해주면 됨.

**Airflow 비번 찾기**:
- `docker compose logs airflow | grep -i password` 를 하면 password 확인할 수 있지만
  ➡️ `airflow | standalone | Password for the admin user has been previously generated in /opt/airflow/simple_auth_manager_passwords.json.generated. Not echoing it here.` 라고 뜰 때가 있음
  ➡️ 이럴 경우엔, `docker compose exec airflow sh -c 'cat $AIRFLOW_HOME/simple_auth_manager_passwords.json.generated` 라고 하면 {admin:password} 형태로 출력됨.

**날짜 설정**:
- `import pendulum` 를 사용하여 UTC 기준이 아니라 DAG 시간을 서울 기준으로 변경함.
