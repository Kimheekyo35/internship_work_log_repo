## 🛍️ 올리브영 글로벌 타사 제품 크롤링

### 문제점
1. playwright으로 했을 때 올리브영 글로벌의 경우, 국가 설정 클릭이 불가능했음
2. seleniumbase로 설정했을 때 region 선택에서 나라를 구분하고 클릭이 제대로 작동하지 않았음. `#headCurrency > li:nth-child(i)`에서 li의 개수가 dropdown했을 때 나오는 개수보다 많았음.

### 해결 방법
1. 우선 올리브영의 경우, CAPTCHA가 자주 나오는 웹 사이트라서 seleniumbase 를 사용해서 크롤링을 진행하였음.

ℹ️ seleniumbase 로 쿠키에 접근할 때 코드

```
def apply_country_cookie(sb: SB, country_cookie_value: str, currency_cookie_value: str) -> None:
    sb.driver.add_cookie(
        {
            "name": "dlvCntry",
            "value": country_cookie_value,
            "domain": "global.oliveyoung.com",
            "path": "/",
        }
    )
    sb.driver.add_cookie(
        {
            "name": "currency",
            "value": currency_cookie_value,
            "domain": "global.oliveyoung.com",
            "path": "/",
        }
    )
    log(f"[cookie] dlvCntry={country_cookie_value} currency={currency_cookie_value}")
    sb.refresh()
    sb.wait_for_ready_state_complete(timeout=DEFAULT_TIMEOUT_MS // 1000)
    sb.sleep(2)
```

해당 코드가 사이트마다 접속해서 쿠키를 적용한 코드임. 영국의 경우  country_cookie_value에 170 을 부여했고, 독일의 경우 90을 부여함.

```
CHANNEL_COUNTRY_COOKIE_MAP: dict[str, str] = {
    "other_l": "170",
    "other_p": "90",
}
CHANNEL_CURRENCY_COOKIE_MAP: dict[str, str] = {
    "other_l": "GBP",
    "other_p": "EUR",
}
```
`CHANNEL_CURRENCY_COOKIE_MAP` 은 화폐 단위임. 영국 -> GBP, 독일 -> EUR 이기 때문.


### ⭐ 이 방법으로 해결한 이유
- '나라를 선택하면 불러오는 정보가 따로 있을텐데, 애초에 접근을 그 부분으로 하면 어떨까?' 라는 생각에서 시작하게 됨. GPT에 물어보니 내가 접근하고자 하는 URL에 해당 쿠키를 적용할 수 있다고 함.

### 쿠키 적용하는 방법

`F12` 버튼을 눌러 DevTools에 들어감 ➡️ Application 클릭 ➡️ cookies ➡️ 도메인 값 ex:) https://global.oliveyoung.com/ ➡️ 원하는 값 선택하여 add_cookie 에 name과 value 넣어주면 됨.
