### Docker 배포

`docker 생성 방법`
1. dockerfile을 만들어줌
```
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt && pip install --upgrade pip

COPY . .

CMD ["python", "main.py"]
EXPOSE 9000
```
정보 작성

2. docker-compose.yml

```
services:
  meeting-room-bot:
    build: .
    container_name: meeting-room-bot
    restart: unless-stopped
    env_file:
      - .env
    volumes:
      - ./credentials.json:/app/credentials.json
      - ./token.json:/app/token.json
credentials, token 같은 json 파일을 도커 파일로 전달해줌 참고할 env도
+ 기타 설정
```

3. docker compose up -d --build 로 실행
4. docker compose logs -f 로 로그 확인
5. 만약 코드를 바꿨을 때 컨테이너를 내리지 않고 재빌드하는 방법
`docker-compose.yml` 에서
``` 
volumes:
  - .:/app
```
추가
작동 중인 터미널은 냅두고 새 터미널에서
`docker compose restart` 하면 끊기지 않고 업데이트 됨
🌟 `requirements.txt` 나 `docker-compose.yml` 을 수정한 경우에는 `docker compose up -d --build` 해줘야함
